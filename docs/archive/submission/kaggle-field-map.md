## What Goes Where

The motivation, what the project is, architecture, Gemma usage, training
iterations, RAG/corpus work, evals, costs, code proof, and limitations all go
inside the Writeup body's `Project Description` editor.

The code/demo/video do not go as prose only. They must also be attached as
`Project Links`, because judges use those links to verify the project.

The images are not a replacement for the writeup. They go in the Media Gallery
and are referenced by the written body implicitly through the narrative.

## Do Not Submit Only

- A short project description.
- A video link without the technical body.
- A GitHub link without the 1,500-word proof-of-work writeup.
- Internal evidence appendices that mention private hosts, passwords, private
  accounts, or consent-check assets.
