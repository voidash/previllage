# PreVillage three-minute video script v1 - Gemma tightened

Deadline: 2026-05-19 05:44 NPT.

Target length: 180 seconds.

Core rule: the video is not a subsystem tour. It should make the viewer feel
the gap, trust the work, understand why Gemma matters, and see the deployment
shape: local office helpdesk, kiosk, WhatsApp, web, official sources, and human
practical knowledge.

## Core line

> I used privilege to find the path. PreVillage exists so the next person does
> not need privilege to use their own government.

## 0:58-1:25 - Why Gemma Matters

Visuals:

- Gemma model terminal/API or Pi local E2B.
- Planner/source-router docs.
- RAG source cards.

Voiceover:

> Gemma was pivotal because this could not just be a cloud chatbot. Government
> help should work where the citizen is: at the counter, on a kiosk, on
> WhatsApp, and eventually inside the office itself.
>
> Gemma gives us an open model small enough for local deployment and capable
> enough to repair noisy speech, understand intent, ask follow-up questions, and
> compose answers from sources.

On-screen:

```text
open model
local enough for offices
grounded enough to ask before answering
```

## 1:25-1:52 - Navigator, Not Chatbot

Visuals:

- `/chat` or `/kiosk` with ambiguous question.
- Follow-up checklist appears.
- Source-backed Jiri/contact answer appears.

Voiceover:

> A normal chatbot guesses. PreVillage does intake.
>
> If a citizenship question needs district, ward, and case type, it asks. If it
> knows the office or contact, it cites the source. If the source is missing, it
> says so.

On-screen:

```text
ask first
retrieve second
answer with sources
```

## 2:12-2:40 - Voice, Kiosk, WhatsApp

Visuals:

- 3-4 people trying kiosk.
- ASR transcript.
- TTS answer playback.
- WhatsApp demo with private details hidden.

Voiceover:

>

>
> If the system cannot know, it can hand the gap to a human officer instead of
> hallucinating.

On-screen:

```text
speak -> fix -> plan -> retrieve -> answer
```
